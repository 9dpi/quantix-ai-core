# Fixed Risk/Reward Rule - AUTO v0

## Rule Specification

**Effective Date**: 2026-01-30  
**Version**: AUTO v0  
**Status**: Active

---

## Parameters

| Parameter | Value | Unit |
|-----------|-------|------|
| **Take Profit (TP)** | 10 | pips |
| **Stop Loss (SL)** | 10 | pips |
| **Risk/Reward Ratio** | 1:1 | Fixed |

---

## Application Scope

This rule applies to **100% of all signals** generated by the Quantix AI Core.

### Fixed Parameters
The TP and SL values are **FIXED** and do **NOT** change based on:
- ❌ Market condition (trending, ranging, volatile)
- ❌ Confidence level (75%, 85%, 95%+)
- ❌ Volatility (ATR, Bollinger Bands)
- ❌ Timeframe (M1, M5, M15, H1, etc.)

---

## Implementation

### Code Location
File: `Quantix_AI_Core/backend/quantix_core/engine/continuous_analyzer.py`

### Logic
```python
# FIXED RISK/REWARD RULE (AUTO v0)
FIXED_TP_PIPS = 0.0010  # 10 pips
FIXED_SL_PIPS = 0.0010  # 10 pips

if direction == "BUY":
    tp = entry + FIXED_TP_PIPS
    sl = entry - FIXED_SL_PIPS
else:  # SELL
    tp = entry - FIXED_TP_PIPS
    sl = entry + FIXED_SL_PIPS

rrr = 1.0  # Fixed 1:1 Risk/Reward Ratio
```

## Rationale

### Why Fixed TP/SL?

1. **Simplicity**: Easy to understand and implement
2. **Consistency**: All signals have the same risk profile
3. **Backtesting**: Uniform parameters for performance evaluation
4. **Risk Management**: Clear, predictable risk per trade

### Trade-offs

**Advantages**:
- ✅ Simple risk calculation
- ✅ Consistent position sizing
- ✅ Easy to automate

**Disadvantages**:
- ⚠️ May not capture full trend moves
- ⚠️ May be too tight in volatile conditions
- ⚠️ May be too wide in ranging markets
